Relay is a distributed task/render-farm orchestrator: three Go binaries (relay-server =
HTTP + gRPC + scheduler with an embedded React/Vite SPA; relay-agent worker that holds one
gRPC stream and runs each task as a subprocess; the relay CLI), plus an MCP server and a
Perforce source-provider. The hardening phase is complete - the 2026-06-10 review's
concurrency/correctness findings are all closed, and the cross-cutting invariants (epoch
fencing on every task-status write, one bounded sender per gRPC stream, identity-checked
teardown, the single job-spec pipeline, no interior pointers across locks, the single JSON
entry point) are enforced and well-tested. The active phase is the embedded web SPA
build-out, now governed by the `design_handoff_relay_holo` "Holo" design direction - a glassy
dark UI spanning the full surface: auth, jobs, workers, schedules, admin, and profile. Auth,
the shell, the Workers/Jobs/Schedules lists, and a read-only Worker detail page have shipped;
the remaining front end - worker write-actions, the job and schedule detail pages, the
full-screen task log, the Admin and Profile consoles, job submit/cancel/retry, and the jobs
filter/lane/timeline views - is now fully captured as backlog items. The small backend endpoints
several of them need (invites/sessions list, job retry, SSE task-log publishing, enrollment revoke)
are filed under the Backend API & server enablers theme; the remaining MCP, agent, and store/schema
items fill in behind the SPA work.
