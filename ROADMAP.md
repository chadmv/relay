# Roadmap

> Generated by /roadmap on 2026-06-21 (refresh, autopilot batch item 4). Source of truth is
> docs/backlog/ - edit items there, then re-run /roadmap. Manual edits may be overwritten.

Relay is a distributed task/render-farm orchestrator: three Go binaries (relay-server =
HTTP + gRPC + scheduler with an embedded React/Vite SPA; relay-agent worker that holds one
gRPC stream and runs each task as a subprocess; the relay CLI), plus an MCP server and a
Perforce source-provider. The current phase is hardening. A 2026-06-10 internal codebase
review surfaced a class of concurrency/correctness bugs that sidestep the cross-cutting
invariants now documented in CLAUDE.md - epoch fencing on every task-status write, one
bounded sender per gRPC stream, identity-checked teardown, the single job-spec pipeline,
no interior pointers across locks, and the single JSON entry point. The team is closing
those findings one at a time; recent sessions shipped the requeue/retry epoch bump, the
stale-stream-teardown fix, the nil-provider source guard, cron source-spec persistence, the
Perforce registry race fix, the runner pipe-drain hang fix, the job status recompute race
fix, the forced-cancel send-backpressure fix, the archived-user offboarding fix, the
background-sweeper Prepare TOCTOU fix, the MCP structured-error delivery fix, the CLI
admin/profile stdout fix, and the cancel/disable concurrent-fan-out fix, with priority on
correctness of the thing being operated - task dispatch, scheduling, and source workspaces -
before broadening features. A secondary workstream builds out the embedded web SPA (auth,
Workers, Jobs, and Schedules lists shipped; detail pages, list filters, and Admin/Profile
still pending). MCP, CLI, and store/schema work fill in behind those two.

## Recently shipped

- [Schedules pagination footer lacks absolute X-Y range](docs/backlog/closed/bug-2026-06-21-schedules-pagination-footer-absolute-range.md) - closed 2026-06-21; `SchedulesPage` replaced its `SHOWING N OF total` page-count footer with an absolute `X-Y of total` range. Added a `startOffset` + `offsets[]` accumulator mirroring its `cursorStack` depth, extracted `goNext`/`goPrev` handlers that mutate all three in lockstep via plain setters (and `chooseSort` resets them), and rendered `{x}-{y} of {total}` (plain hyphen) via the shared `computePageRange` helper, advancing by actual page size so partial last pages stay exact. Both buttons stay gated on `isPlaceholderData` so the stacks cannot desync. Three new tests (first page, partial last, page-back) RED against the old footer; full web suite (172) + tsc green; review found no high/medium/low issues.
- [Add cwd assertions to the three perforce unit tests](docs/backlog/closed/idea-2026-05-01-add-cwd-assertions-perforce.md) - closed 2026-06-21; the three `TestProvider_*` unit tests pinned the `-c <client>` argv half of the client-selection contract but covered the cwd half only implicitly. Added a shared `assertCwdContract(t, fr, wsRoot)` helper that requires workspace-scoped `-c <client>` calls to run from wsRoot and global calls (`ResolveHead`, `client -o/-i/-d`) to run with empty cwd, asserted across every recorded call. Proven non-vacuous (breaking the production sync cwd fails all three). Test-only; no production change.
- [owner_email lookup errors are swallowed silently](docs/backlog/closed/bug-2026-06-05-owner-email-lookup-errors.md) - closed 2026-06-21; `fillOwnerEmails` on the admin schedules-list path batch-resolved `owner_email` via `GetUserEmailsByIDs` and returned silently on error, leaving the field empty with no log line - invisible to operators. Added a `log.Printf` (stdlib, matching the project's `subsystem:`-prefixed convention) recording the id count and the error; best-effort behavior otherwise unchanged. One-line observability fix; verified by build + vet + the api suite.
- [Cancel/disable handlers send synchronously to workers](docs/backlog/closed/bug-2026-06-10-cancel-disable-handlers-send-synchronously.md) - closed 2026-06-21; the cancel (`handleCancelJob`) and disable (`handleDisableWorker`) handlers looped over a job's/worker's running tasks and called `registry.Send` sequentially after the DB commit, so N tasks on one wedged worker cost up to N x ~5s in the request path. Extracted a shared `(*Server).sendCancelSignals` helper that fans the best-effort `CancelTask` signals out concurrently via a `sync.WaitGroup` and waits for all, bounding the caller to ~one send timeout. New unit test `TestSendCancelSignals_FanOutIsConcurrent` (RED against a sequential helper, GREEN after the fan-out); adversarial review re-verified `registry.Send`/`workerSender.Send` concurrency safety. No high/medium/low findings.
- [Bump GitHub Actions off deprecated Node 20](docs/backlog/closed/idea-2026-06-03-bump-github-actions-deprecated-node.md) - closed 2026-06-21; bumped all Node-20 action majors to their Node-24 successors across every workflow - `actions/checkout@v4`->`@v5`, `actions/setup-python@v5`->`@v6`, `actions/setup-go@v5`->`@v6` - covering `release.yml`, `python.yml`, and `go-ci.yml` (added after the item was filed). The Docker-based `pypa/gh-action-pypi-publish@release/v1` is not a Node action and was left unchanged. Verified by grep (no Node-20 pins remain) plus a YAML parse of all three workflows.
- [Paginate revoked workers list (UI and client)](docs/backlog/closed/bug-2026-06-05-paginate-revoked-workers-list.md) - closed 2026-06-21; added cursor pagination to the revoked-workers list mirroring JobsPage, with an `X-Y of total` footer and a shared `web/src/lib/pageRange.ts`. Adversarial review returned no high/medium findings; full web suite + `tsc` green.
- [Jobs pagination footer lacks absolute X-Y range](docs/backlog/closed/bug-2026-06-05-jobs-pagination-footer-absolute-range.md) - closed 2026-06-21; the Jobs footer now shows an absolute `X-Y of total` range via a running `offsets` stack, with offset mutations gated behind the same `isPlaceholderData` button guard as the cursor stack. Range math extracted to a pure `computePageRange` helper.
- [useJobs and useJobStats share query-key prefix](docs/backlog/closed/bug-2026-06-05-usejobs-usejobstats-query-key-prefix.md) - closed 2026-06-21; `useJobStats`'s react-query key changed to a distinct top-level `['job-stats']`, so a broad `invalidateQueries({ queryKey: ['jobs'] })` no longer clobbers the stats query.
