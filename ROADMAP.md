# Roadmap

> Generated by /roadmap on 2026-06-21 (refresh, autopilot). Source of truth is docs/backlog/ -
> edit items there, then re-run /roadmap. Manual edits to this file may be overwritten.

Relay is a distributed task/render-farm orchestrator: three Go binaries (relay-server =
HTTP + gRPC + scheduler with an embedded React/Vite SPA; relay-agent worker that holds one
gRPC stream and runs each task as a subprocess; the relay CLI), plus an MCP server and a
Perforce source-provider. The current phase is hardening. A 2026-06-10 internal codebase
review surfaced a class of concurrency/correctness bugs that sidestep the cross-cutting
invariants now documented in CLAUDE.md - epoch fencing on every task-status write, one
bounded sender per gRPC stream, identity-checked teardown, the single job-spec pipeline,
no interior pointers across locks, and the single JSON entry point. The team is closing
those findings one at a time; the last several sessions shipped the requeue/retry epoch
bump, the stale-stream-teardown fix, the nil-provider source guard, cron source-spec
persistence, the Perforce registry race fix, the runner pipe-drain hang fix, the job
status recompute race fix, the forced-cancel send-backpressure fix, the archived-user
offboarding fix (token rejection + schedule disable), and the background-sweeper Prepare
TOCTOU fix, with priority on correctness of the
thing being operated - task dispatch, scheduling, and source workspaces - before
broadening features. A secondary workstream builds out the embedded web SPA (auth,
Workers, Jobs, and Schedules lists shipped; detail pages, list filters, and Admin/Profile
still pending). MCP, CLI, and store/schema work fill in behind those two.

- [Background sweeper has a Prepare TOCTOU on the dominant eviction path](docs/backlog/closed/bug-2026-06-21-sweeper-prepare-toctou.md) - closed 2026-06-21; the background sweeper now claims `p.evicting[shortID]` across its slow per-entry evict via a new optional `Sweeper.Claim` hook wired to the new `Provider.ReserveForEvict` (background sweeper only; the internal `EvictWorkspace` Sweeper keeps `Claim` nil). `Prepare`'s existing post-`Acquire` re-check now observes a sweeper claim and backs out, so a live `Prepare` never syncs into a workspace `SweepOnce`->`evict` is deleting; exactly one of {sweep, prepare} proceeds. A lost claim is a benign `ErrEvictClaimLost` sentinel `SweepOnce` skips without logging or counting. Deterministic timing-free regression test driving a concurrent `SweepOnce` into the Prepare gap, proven RED before the fix. relay-verify returned no high/medium findings; two low maintainability notes addressed with cross-reference comments on the twin reservation blocks.
- [Provider.EvictWorkspace leaks per-task state and has a lock TOCTOU](docs/backlog/closed/bug-2026-06-20-evictworkspace-state-leak-toctou.md) - closed 2026-06-21; `EvictWorkspace`'s ad-hoc Sweeper now wires `OnEvictedCB: p.InvalidateWorkspace` (per-task `syncedPaths` no longer survives a manual eviction), and the Prepare TOCTOU is closed via a `p.evicting` reservation set (claimed under `p.mu` after an inline holder check, slow evict run lock-free, cleared in a defer) plus a post-`Acquire` re-check in `Prepare` that backs out if a reservation landed in the pre-check->Acquire gap. Exactly one of {evict, prepare} wins; the loser does no harm. relay-verify caught a residual pre-`Acquire` window in the first attempt; the re-check closed it (proven load-bearing). The same TOCTOU class on the background sweeper was split into a separate item.
- [Agent token file is world-readable on Windows](docs/backlog/closed/bug-2026-06-10-agent-token-windows-acl.md) - closed 2026-06-21; on Windows the agent token file (`%ProgramData%\relay\token`) now gets a PROTECTED (non-inherited) DACL via the SDDL `D:PAI(A;;FA;;;OW)(A;;FA;;;SY)(A;;FA;;;BA)` - full control only to Owner Rights, LocalSystem, and Administrators - applied with `SetNamedSecurityInfo`+`PROTECTED_DACL_SECURITY_INFORMATION`, severing the inherited ProgramData `BUILTIN\Users` ACE that let any local user steal the bearer token. Build-tag split (no-op on Unix, where 0600 is correct). `Persist` hard-fails and removes the file on an ACL error.
