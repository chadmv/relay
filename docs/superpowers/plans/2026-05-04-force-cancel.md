# Force Cancel & Process-Tree Kill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `relay cancel` kill the entire subprocess tree by default (eliminating leaked grandchildren) and add `--force` for impatient cancellation that skips workspace finalize and pipe drain.

**Architecture:** Tree-kill is implemented via OS-native primitives — process groups on Unix (`Setpgid` + `kill -PGID`) and Job Objects on Windows (`AssignProcessToJobObject` + `TerminateJobObject`). A new `bool force` field on the `CancelTask` proto plumbs the flag from CLI → REST → gRPC → agent. The agent runner gains a `forced atomic.Bool` consulted at cancel time to skip pipe drain (by closing stdout/stderr pipe handles directly) and at the workspace-finalize defer to skip cleanup.

**Tech Stack:** Go 1.22+, `os/exec` (Go 1.20+ `cmd.Cancel`/`cmd.WaitDelay`), `syscall.SysProcAttr.Setpgid` (Unix), `golang.org/x/sys/windows` (Job Objects), Protobuf, gRPC, stdlib `flag` package, sqlc, pgx.

**Spec:** [docs/superpowers/specs/2026-05-04-force-cancel-design.md](../specs/2026-05-04-force-cancel-design.md)

```go
// setStepPipes records the pipe handles for the currently executing step so
// the cmd.Cancel callback can force-close them on a forced cancel.
func (r *Runner) setStepPipes(stdout, stderr io.Closer) {
	r.stepPipes.mu.Lock()
	r.stepPipes.stdout = stdout
	r.stepPipes.stderr = stderr
	r.stepPipes.mu.Unlock()
}

// clearStepPipes drops the references after a step completes. Safe to call
// even if setStepPipes wasn't called (no-op).
func (r *Runner) clearStepPipes() {
	r.stepPipes.mu.Lock()
	r.stepPipes.stdout = nil
	r.stepPipes.stderr = nil
	r.stepPipes.mu.Unlock()
}

// closeStepPipesForForce closes the live pipe handles, unblocking pipeLog
// readers immediately. Called from the cmd.Cancel callback only when forced.
func (r *Runner) closeStepPipesForForce() {
	r.stepPipes.mu.Lock()
	stdout, stderr := r.stepPipes.stdout, r.stepPipes.stderr
	r.stepPipes.mu.Unlock()
	if stdout != nil {
		_ = stdout.Close()
	}
	if stderr != nil {
		_ = stderr.Close()
	}
}
```

//go:build !windows

package agent

import (
	"os/exec"
	"syscall"
)

// setupProcTree configures cmd so that:
//   - the child starts in its own process group (Setpgid), so all descendants
//     inherit the same PGID by default.
//   - cmd.Cancel (invoked when the context tied to exec.CommandContext is
//     cancelled) sends SIGKILL to the entire process group, killing every
//     descendant in one shot.
//   - if r.forced is set at cancel time, the runner's stdout/stderr pipe
//     handles are closed immediately, unblocking pipeLog readers without
//     waiting on the kernel pipe-buffer drain bounded by cmd.WaitDelay.
//
// cmd.WaitDelay is set unconditionally to the existing 5s budget; it bounds
// pipe-reader unblocking in the non-forced path.
func setupProcTree(cmd *exec.Cmd, r *Runner) {
	if cmd.SysProcAttr == nil {
		cmd.SysProcAttr = &syscall.SysProcAttr{}
	}
	cmd.SysProcAttr.Setpgid = true

	cmd.Cancel = func() error {
		pid := 0
		if cmd.Process != nil {
			pid = cmd.Process.Pid
		}
		// Negative PID targets the process group whose PGID == |pid|.
		// We set Setpgid above so PGID == child PID.
		if pid > 0 {
			_ = syscall.Kill(-pid, syscall.SIGKILL)
		}
		if r.forced.Load() {
			r.closeStepPipesForForce()
		}
		return nil
	}
}
```

//go:build windows

package agent

import (
	"log"
	"os/exec"
	"sync"

	"golang.org/x/sys/windows"
)

// setupProcTree configures cmd so that:
//   - the child is assigned to a Windows Job Object with
//     JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE set, so any descendant the child
//     spawns is also tied to the job and dies with it.
//   - cmd.Cancel calls TerminateJobObject, which kills every process in the
//     job atomically — direct child and any descendants.
//   - if r.forced is set at cancel time, stdout/stderr pipe handles are
//     closed immediately, bypassing the pipe-buffer drain bounded by
//     cmd.WaitDelay.
//
// The Job Object handle is closed in a deferred wrapper around cmd.Cancel,
// triggered after cmd.Wait returns. KILL_ON_JOB_CLOSE is a safety net: if the
// agent process itself crashes without closing the handle, the kernel still
// reaps the descendants when our process exits.
func setupProcTree(cmd *exec.Cmd, r *Runner) {
	var (
		jobMu sync.Mutex
		job   windows.Handle
		setup bool
	)

	// We have to defer the Job Object creation and assignment until after
	// cmd.Start runs, because we need cmd.Process.Handle. Go's exec package
	// does not give us a post-Start hook, so we attach a Cancel that, on
	// first invocation, ensures the Job Object exists and the child is in it.
	// The "happy path" assignment is performed eagerly via a goroutine that
	// polls cmd.Process; in practice cmd.Process is set synchronously by
	// cmd.Start, so the polling completes immediately.
	ensureAssigned := func() {
		jobMu.Lock()
		defer jobMu.Unlock()
		if setup {
			return
		}
		setup = true
		if cmd.Process == nil {
			return
		}
		h, err := windows.CreateJobObject(nil, nil)
		if err != nil {
			log.Printf("agent: CreateJobObject: %v", err)
			return
		}
		var info windows.JOBOBJECT_EXTENDED_LIMIT_INFORMATION
		info.BasicLimitInformation.LimitFlags = windows.JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE
		if _, err := windows.SetInformationJobObject(
			h,
			windows.JobObjectExtendedLimitInformation,
			uintptr(unsafe.Pointer(&info)),
			uint32(unsafe.Sizeof(info)),
		); err != nil {
			log.Printf("agent: SetInformationJobObject: %v", err)
			_ = windows.CloseHandle(h)
			return
		}
		procHandle := windows.Handle(cmd.Process.Pid) // placeholder; see note
		_ = procHandle
		// Open the process handle for the assignment; cmd.Process exposes a
		// *os.Process whose internal Handle is what we want.
		ph, err := windows.OpenProcess(windows.PROCESS_SET_QUOTA|windows.PROCESS_TERMINATE, false, uint32(cmd.Process.Pid))
		if err != nil {
			log.Printf("agent: OpenProcess(%d): %v", cmd.Process.Pid, err)
			_ = windows.CloseHandle(h)
			return
		}
		defer windows.CloseHandle(ph)
		if err := windows.AssignProcessToJobObject(h, ph); err != nil {
			log.Printf("agent: AssignProcessToJobObject: %v", err)
			_ = windows.CloseHandle(h)
			return
		}
		job = h
	}

	cmd.Cancel = func() error {
		ensureAssigned()
		jobMu.Lock()
		h := job
		jobMu.Unlock()
		if h != 0 {
			_ = windows.TerminateJobObject(h, 1)
		} else if cmd.Process != nil {
			// Fallback: if Job Object setup failed, at least kill the direct child.
			_ = cmd.Process.Kill()
		}
		if r.forced.Load() {
			r.closeStepPipesForForce()
		}
		return nil
	}

	// Best-effort: assign the process to the job as soon as Start completes.
	// Caller (Runner.Run) invokes cmd.Start before reading from pipes, so by
	// the time the first read blocks, cmd.Process is set. We hook this via
	// the WaitDelay path: a tiny goroutine spins until cmd.Process != nil
	// and then runs ensureAssigned(). This is safe because ensureAssigned
	// is idempotent.
	go func() {
		for cmd.Process == nil {
			// Cheap busy-wait; cmd.Start sets Process synchronously, so this
			// loop should complete in < 1 microsecond in practice.
		}
		ensureAssigned()
	}()
}
```

// TestRunner_TreeKill_RealSubprocesses spawns a parent script that forks a
// long-running grandchild and writes the grandchild's PID to a file.
// After cancel, we verify the grandchild process is gone within 2s.
func TestRunner_TreeKill_RealSubprocesses(t *testing.T) {
	dir := t.TempDir()
	pidFile := filepath.Join(dir, "grandchild.pid")

	var script string
	var argv []string
	switch runtime.GOOS {
	case "windows":
		// PowerShell: spawn a detached ping and write its PID to a file.
		script = "$p = Start-Process -PassThru -WindowStyle Hidden ping -ArgumentList '127.0.0.1','-n','60'; " +
			"$p.Id | Out-File -Encoding ASCII '" + pidFile + "'; " +
			"Wait-Process -Id $p.Id"
		argv = []string{"powershell", "-NoProfile", "-Command", script}
	default:
		// Bash: fork sleep, record PID, then sleep ourselves.
		script = "sleep 60 & echo $! > " + pidFile + "; wait"
		argv = []string{"sh", "-c", script}
	}

	sendCh := make(chan *relayv1.AgentMessage, 256)
	task := &relayv1.DispatchTask{
		TaskId:   "t-tree",
		JobId:    "j-tree",
		Commands: singleCmd(argv),
	}

	r, runCtx := newRunner(task.TaskId, task.Epoch, sendCh, context.Background(), 0)

	done := make(chan struct{})
	go func() { defer close(done); r.Run(runCtx, task) }()

	// Wait for the grandchild PID to appear in the file.
	var grandPID int
	deadline := time.Now().Add(5 * time.Second)
	for time.Now().Before(deadline) {
		data, err := os.ReadFile(pidFile)
		if err == nil {
			pidStr := strings.TrimSpace(string(data))
			if pidStr != "" {
				p, perr := strconv.Atoi(pidStr)
				if perr == nil && p > 0 {
					grandPID = p
					break
				}
			}
		}
		time.Sleep(50 * time.Millisecond)
	}
	require.NotZero(t, grandPID, "grandchild PID file never appeared")
	t.Logf("grandchild PID: %d", grandPID)

	// Cancel (default — exercises tree-kill).
	r.Cancel(false)

	select {
	case <-done:
	case <-time.After(8 * time.Second):
		t.Fatal("runner did not exit within 8s after cancel")
	}

	// Poll for grandchild gone within 2s.
	gone := false
	pollDeadline := time.Now().Add(2 * time.Second)
	for time.Now().Before(pollDeadline) {
		if !pidAlive(grandPID) {
			gone = true
			break
		}
		time.Sleep(100 * time.Millisecond)
	}
	if !gone {
		// Best-effort cleanup so we don't leak the process from a failing test.
		_ = killPID(grandPID)
		t.Fatalf("grandchild PID %d still alive 2s after cancel — tree-kill did not propagate", grandPID)
	}
}

// pidAlive returns true if a process with the given PID is alive. Cross-platform.
func pidAlive(pid int) bool {
	if runtime.GOOS == "windows" {
		// tasklist returns zero-rows output if the PID is gone.
		out, err := exec.Command("tasklist", "/FI", fmt.Sprintf("PID eq %d", pid), "/NH").Output()
		if err != nil {
			return false
		}
		return strings.Contains(string(out), strconv.Itoa(pid))
	}
	// Unix: signal 0 probes existence without delivering a signal.
	proc, err := os.FindProcess(pid)
	if err != nil {
		return false
	}
	return proc.Signal(nil /* signal 0 */) == nil
}

func killPID(pid int) error {
	if runtime.GOOS == "windows" {
		return exec.Command("taskkill", "/F", "/PID", strconv.Itoa(pid)).Run()
	}
	proc, err := os.FindProcess(pid)
	if err != nil {
		return err
	}
	return proc.Kill()
}
```

Note: on Unix, `proc.Signal(nil)` doesn't compile — use `syscall.Signal(0)` instead. Adjust the import and call:

```go
import "syscall"
// ...
return proc.Signal(syscall.Signal(0)) == nil
```

#### `relay cancel`

Cancel a job. Pending and queued tasks are marked failed immediately. For tasks already running on an agent, the agent terminates the entire subprocess tree (the direct child and any descendants), then runs workspace cleanup before reporting the task as failed. The agent's brief pipe-drain budget (5 s) gives the subprocess one last chance to flush stdout/stderr to the relay log.

Use `--force` to skip pipe drain and workspace cleanup so the agent is freed as quickly as possible. This is the right choice when a task is genuinely stuck or you don't care about the last few KB of log output. Forced cancel may leave a Perforce workspace in a partial state; the next sync on that worker treats it as a cold target.

```sh
relay cancel <job-id>           # default: tree-kill, drain logs, cleanup workspace
relay cancel <job-id> --force   # tree-kill, skip drain, skip cleanup
```
```
